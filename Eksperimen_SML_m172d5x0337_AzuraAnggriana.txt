1. INFORMASI DATASET
Nama Dataset      : Fraud Detection (20.000 sampel)
Sumber Dataset    : Kaggle (versi disederhanakan)
Jumlah Fitur      : 29 fitur numerik (V1–V28 + Amount)
Target            : Class (0 = normal, 1 = fraud)
Preprocessing     :
 - Drop duplikasi
 - Train-test split (80:20)
 - StandardScaler
 - Menyimpan data ke folder: fraud_detection_preprocessing
   • fraud_train_preprocessed.csv
   • fraud_test_preprocessed.csv

2. EKSPERIMEN BASELINE
Eksperimen MLflow : FraudDetection_Baseline
Model Baseline    : Logistic Regression
Run Name          : logreg_baseline

METRIK:
 - Accuracy       : ± 0.596
 - Precision      : ± 0.589
 - Recall         : ± 0.579
 - ROC-AUC        : ± 0.60
 - F1-Score       : ± 0.584

ARTIFACTS:
 - training_confusion_matrix.png
 - training_precision_recall_curve.png
 - training_roc_curve.png
 - model.pkl
 - MLmodel
 - requirements.txt
 - python_env.yaml

CATATAN:
Model baseline menghasilkan performa rendah karena:
 - data sangat imbalance
 - logistic regression sederhana
 - belum ada tuning parameter

3. EKSPERIMEN TUNING (SKILLED)
Eksperimen MLflow : FraudDetection_Tuning

3.1 Logistic Regression – Tuning
Run Name : logreg_tuning

Parameter:
 - C = 0.1
 - solver = liblinear
 - class_weight = balanced
 - max_iter = 500

METRIK:
 - Accuracy       : ± 0.71
 - Precision      : ± 0.66
 - Recall         : ± 0.77
 - ROC-AUC        : ± 0.78
 - F1-Score       : ± 0.71

ARTIFACTS:
 - training_confusion_matrix.png
 - training_precision_recall_curve.png
 - training_roc_curve.png
 - model.pkl
 - MLmodel
 - conda.yaml
 - python_env.yaml

CATATAN:
Terdapat peningkatan signifikan terutama pada recall dan auc-score,
model lebih sensitif menangkap fraud.

3.2 Random Forest – Tuning
Run Name : random_forest_tuning

Parameter:
 - n_estimators = 200
 - max_depth = 10
 - min_samples_split = 5
 - class_weight = balanced_subsample

METRIK:
 - Accuracy       : ± 0.73
 - Precision      : ± 0.70
 - Recall         : ± 0.75
 - ROC-AUC        : ± 0.80
 - F1-Score       : ± 0.72

ARTIFACTS:
 - training_confusion_matrix.png
 - training_precision_recall_curve.png
 - training_roc_curve.png
 - model.pkl
 - MLmodel

CATATAN:
Random Forest menghasilkan performa terbaik pada AUC dan recall
dibanding Logistic Regression tuning.

4. PERBANDINGAN AKHIR
Model terbaik berdasarkan metrik AUC & Recall:
 → **Random Forest (tuning)**

Alasan:
 - mampu menangani nonlinearitas
 - stabil pada data imbalance
 - mampu capture pattern fraud yang kompleks
 - memiliki recall tinggi (lebih penting untuk fraud detection)
